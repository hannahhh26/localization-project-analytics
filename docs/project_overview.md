# Project Overview
## Background

Localization service providers (LSPs) manage a large number of translation projects simultaneously across multiple clients. They need to coordinate project managers, project types, translators, locales, deadlines, and quality assurance activities. As the number of active projects grows, monitoring operational performance, resource utilisation, and delivery risk becomes increasingly difficult when information is spread across multiple systems or spreadsheets.

This project simulates a localization operations environment using synthetic data to model common business processes and reporting requirements. It provides a realistic dataset that can be analysed using SQL and Python to generate operational insights and support data-driven decision making.

## Objectives

The primary objectives of this project are to:

- Design a relational database representing a simplified localization operations workflow.
- Generate realistic synthetic data for clients, projects, translators, languages, and delivery activities.
- Query operational and performance metrics using SQL.
- Analyse and visualise operational data using Python.
- Demonstrate how business insights can support operational decision making.

## Project Scope

This project simulates the operations side of a localization company using synthetic data.

It includes:
- relational database design
- synthetic data generation
- SQL analysis
- Python data processing
- business reporting

The project does not aim to replicate any real company's internal systems or use proprietary data. All datasets are synthetically generated and have been created solely for portfolio purposes.
